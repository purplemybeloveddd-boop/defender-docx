# Privacy Policy – Defender Bot

**Effective Date:** July 19, 2026

## 1. Introduction

Defender Bot ("Defender", "the Bot", "we", "our", or "us") is a Discord security, moderation, verification, and server management bot. This Privacy Policy explains what information is collected, why it is collected, how it is used, and how it is protected when you use Defender Bot.

By adding or using Defender Bot, you acknowledge and agree to this Privacy Policy.

---

# 2. Information We Collect

To provide Defender's features, the Bot may collect and process the following Discord-related information.

## Server (Guild) Information

* Guild (Server) ID
* Guild name
* Guild icon
* Guild banner
* Guild splash image (if available)
* Guild owner ID
* Guild creation date
* Guild features
* Verification level
* Server boost level
* Community settings
* Other publicly available server metadata provided by the Discord API

---

## User Information

The Bot may collect:

* Discord User ID
* Username
* Global Display Name
* Server Nickname (when applicable)
* Avatar/Profile Picture URL
* Banner URL (if available)
* Bot/User status where necessary for moderation features

Defender does **not** collect passwords, email addresses, payment information, or Discord login credentials.

---

## Role Information

The Bot may collect:

* Role IDs
* Role names
* Role colors
* Role permissions
* Role positions
* Role icons (where available)
* Role metadata

---

## Channel Information

The Bot may collect:

* Channel IDs
* Channel names
* Channel types
* Parent categories
* Channel positions
* Permission overwrites
* NSFW settings
* Slowmode settings
* Topics
* Other configuration required for restoring channels during backups

---

## Emoji & Sticker Information

The Bot may collect:

* Emoji IDs
* Emoji names
* Animated status
* Emoji images
* Sticker IDs
* Sticker names
* Sticker metadata

---

## Webhook Information

When backup or recovery features require it, Defender may collect:

* Webhook IDs
* Webhook names
* Webhook channels

Webhook tokens are not retained unless required for an active feature.

---

## Backup Metadata

When using backup features, Defender may securely store information including:

* Server structure
* Categories
* Channels
* Roles
* Permissions
* Emojis
* Stickers
* Verification settings
* Moderation configuration
* Anti-Nuke configuration
* Security configuration
* Other server settings necessary for restoration

Backup data exists solely to restore the server if requested by an authorized server owner.

---

## Verification Data

When using Defender's verification system, the Bot may temporarily process:

* User ID
* Guild ID
* Verification status
* Verification timestamp
* Verification method
* Verification session identifiers

This information is used only to determine whether a member has successfully completed verification.

---

## Subscription Information

For subscription management, Defender may store:

* Guild ID
* Subscription status
* Subscription tier
* Activation/license key
* Subscription expiration date
* Purchase or activation timestamps
* Feature entitlement information

Payment information is **not** stored by Defender if payments are handled by an external payment provider.

---

# 3. Why We Collect Data

The information collected is used only to provide Defender's functionality, including:

* Server protection
* Anti-Nuke systems
* Moderation features
* Security monitoring
* Server verification
* Role management
* Backup and restoration
* Subscription management
* License validation
* Feature configuration
* Security logging
* Abuse prevention
* Improving reliability and stability

---

# 4. Data Sharing

Defender does **not** sell, rent, or trade user or server data.

Information may only be disclosed:

* When required by applicable law.
* To investigate abuse of the service.
* To protect the integrity or security of Defender.
* When necessary to operate services provided by trusted infrastructure providers.

---

# 5. Data Storage

Collected information may be stored securely in databases or encrypted storage systems operated by Defender.

Reasonable technical and organizational measures are used to help protect stored information against unauthorized access, alteration, or disclosure.

---

# 6. Data Retention

Data is retained only for as long as reasonably necessary to operate Defender.

This may include:

* Active subscription records
* Backup data until deleted or expired
* Verification records for operational purposes
* Configuration data while the Bot remains in the server

Data may be deleted after the Bot is removed from a server, a subscription expires (where appropriate), or upon request, unless retention is required for legal or security reasons.

---

# 7. Third-Party Services

Defender operates using Discord's API and may rely on trusted third-party infrastructure providers for hosting, databases, analytics, or storage.

These providers may process limited information necessary to operate Defender's services.

---

# 8. Your Rights

Server owners may request:

* Deletion of stored backup data
* Deletion of subscription information where applicable
* Removal of stored configuration data after removing the Bot
* Information regarding stored server data, where technically feasible

---

# 9. Children's Privacy

Defender is intended for use by Discord users in accordance with Discord's Terms of Service. Defender does not knowingly collect personal information from individuals where prohibited by applicable law.

---

# 10. Changes to This Policy

This Privacy Policy may be updated periodically.

Material changes may be announced through Defender's support server, documentation, or official announcement channels.

Continued use of Defender after changes become effective constitutes acceptance of the updated Privacy Policy.

---

# 11. Contact

If you have questions regarding this Privacy Policy, data handling practices, or wish to request deletion of eligible stored data, please contact the Defender development team through the official support server or official support channels.

---

**Summary**

Defender only collects the Discord data necessary to provide its security, moderation, verification, subscription, licensing, and backup features. Defender does not collect Discord passwords, email addresses, or payment credentials, and does not sell user or server data.
